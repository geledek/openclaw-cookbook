# OpenFang：从 Agent 能力到 Agent 组织

## Core Concept: "Hands"

Traditional agents = outsourced workers. You say something, they do one thing, flow breaks, you reconnect.

**Hands** = employees with complete SOPs. You give the goal, they run the plan autonomously, know when to do what, which tools to use, where to deliver results. No babysitting.

The shift: from talking about agent **capabilities** to talking about agent **work patterns**.

## Built-in Hands

| Hand | Function |
|---|---|
| **Collector** | Monitor competitors + sentiment, auto-build knowledge graphs |
| **Lead** | Prospect mining, scoring, dedup, packaged output |
| **Researcher** | Cross-validated reports with citations |
| **Clip** | Auto video editing, subtitles, cover images, publishing |
| **Browser** | Web automation, mandatory human confirmation for payments |

Key: not "can it use tools" but "can it close the loop on an entire workflow."

## Security Architecture

16-layer security built into the architecture:
- WASM sandbox
- Hash-chain audit logs
- Mandatory human confirmation for spending/payments
- Signals "production-grade system" not a toy

## Agent Evolution Thesis

1. **OpenClaw** opened capabilities (what agents can do)
2. **ZeroClaw** polished performance (how fast/cheap)
3. **OpenFang** starts building organization (how workflows standardize)

Docker analogy: Docker didn't make programs smarter, it made deployment standardized and reproducible. Hands does the same for AI workflows.

## Future Competition

Not who has the stronger model, but whose agent workflows run more stably, more safely, at lower cost. When one person manages 10-50-100 autonomous workflows, they're managing a digital team.
